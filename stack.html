INSTRUCTIONS

How do we work with Stack of Queues?


We are working with at most 20 elements here.


Enter the number of queues (preferebaly divisible by 20)


Insert the elements in the order of bottom to top.


If we want to enqueue in queue2 while queue1 is empty an alert message is throwed.


If we want to dequeue in queue3 while queue4 is not empty an alert message is throwed


Pay attention to the alerts when you make a mistake. If you understand the properties of stack of queues you will not face any constraint alerts.

OPERATIONS

PUSH()

PUSH()

inserts elements into the stack. If the stack is full then the overflow condition occurs.

ENQUEUE()

ENQUEUE()

Adds an element to the end of the queue.

POP()

POP()

removes elements from the stack. If the stack is empty, this state is known as an underflow state.

DEQUEUE()

DEQUEUE()

Removes an element from the front of the queue

isFULL()
/
isEMPTY()

isFULL() & isEMPTY

isFull() operation checks whether the stack/queue is full and isEmpty() operation verifies whether the stack/queue is empty.

PEEK()

PEEK()

acquires the data element available at the front of the queue/top of the stack without deleting it.
